## Опис виконаної роботи

У межах лабораторної роботи було проведено комплексний аналіз та рефакторинг legacy-коду обчислювальних функцій фінансового обліку за допомогою AI (ChatGPT/Claude). 

### Що було виявлено (AI Code Review):
1. **Низька читаність коду:** Використання назв на кшталт `calc`, `d`, `r`, `m` замість семантично зрозумілих назв бізнес-сутностей.
2. **Застарілі стандарти:** Використання `var`, що створює ризики порушення областей видимості.
3. **Проблеми з продуктивністю:** Створення екземплярів `new Date()` всередині ітераційних циклів.
4. **Відсутність типізації:** Повна відсутність валідації вхідних структур, що загрожує падінням програми в рантаймі при отриманні `undefined` полів.

Детальний звіт та критична оцінка рекомендацій ШІ знаходяться у файлі `ai-review.md`.

### Що було виправлено (Рефакторинг):
* Код повністю переведено на **TypeScript**. Створено інтерфейси `Transaction` та типи `TransactionType`, `CategorySummary`.
* Проведено повне перейменування функцій та змінних відповідно до конвенцій Clean Code (`calculateBalance`, `getMonthlyCategorySummary`).
* Код розбито на менші **чисті функції** (виділено хелпер `isTransactionInMonth`).
* Прибрано магічні рядки, додано суворі порівняння (`===`), а застарілий `var` замінено на `const` та `let`.
* Додано стандартизовані **JSDoc-коментарі** для кожної функції.
